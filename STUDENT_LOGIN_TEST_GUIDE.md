## 🎯 测试场景

### 场景一：学员登录机型选择

**测试步骤：**
1. 使用学员账号登录（如：student001 / admin123）
2. 系统检测到用户为学员，自动弹出机型选择对话框
3. 显示该学员可选择的机型列表：
   - 张三学员：AS350 松鼠直升机、R44 雷鸟直升机
4. 选择一个机型（如：AS350）
5. 确认选择，进入系统主界面

**预期结果：**
- 机型选择对话框正确显示
- 只显示该学员授权的机型
- 选择机型后，系统记录当前机型选择
- 后续数据访问受机型权限控制

### 场景二：数据权限验证

**测试步骤：**
1. 学员登录并选择AS350机型
2. 访问"学员管理"页面
3. 查看学员列表数据

**预期结果：**
- 只显示与AS350机型相关的学员数据
- 其他机型的学员数据被过滤掉

### 场景三：切换机型

**测试步骤：**
1. 学员已登录并选择了AS350机型
2. 需要切换到R44机型
3. 调用机型选择API重新选择

**预期结果：**
- 能够成功切换到其他授权的机型
- 数据权限相应更新

### 场景四：权限边界测试

**测试步骤：**
1. 学员尝试选择未授权的机型
2. 验证系统安全控制

**预期结果：**
- 只能选择授权的机型
- 非授权机型不显示或选择时报错

## 🐛 常见问题排查

### 问题1：登录后没有弹出机型选择对话框
**可能原因：**
- 用户不是学员角色
- 前端组件未正确集成
- 接口调用失败

**排查步骤：**
1. 检查用户是否有student_id字段
2. 检查用户角色是否包含"学员"
3. 检查浏览器控制台是否有错误

### 问题2：机型选择后数据权限不生效
**可能原因：**
- DataScope注解未添加
- 数据库字段关联错误
- 切面处理异常

**排查步骤：**
1. 检查Controller方法是否添加@DataScope注解
2. 检查数据库student_machine_type关联表数据
3. 查看后端日志确认切面是否执行

### 问题3：学员看不到任何数据
**可能原因：**
- 机型关联数据缺失
- SQL过滤条件过于严格
- 当前机型选择为空

**排查步骤：**
1. 检查tms_student_machine_type表数据
2. 检查用户current_machine_type_id字段
3. 确认相关业务数据存在

### 学员机型关联
```sql
SELECT s.student_name, mt.machine_type_name, smt.is_primary
FROM tms_student_machine_type smt
JOIN tms_student s ON smt.student_id = s.student_id
JOIN tms_machine_type mt ON smt.machine_type_id = mt.machine_type_id
ORDER BY s.student_id, smt.is_primary DESC;
```

### 培训班次数据
```sql
SELECT tc.class_name, mt.machine_type_name, tc.status
FROM tms_training_class tc
JOIN tms_machine_type mt ON tc.machine_type_id = mt.machine_type_id;
```
